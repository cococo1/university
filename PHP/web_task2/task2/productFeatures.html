Goal description

The goal of this project is to get an online site which is able to create/edit/delete surveys and store all this data on a database.

User roles

We can have the following types of users:

  • Anonymous user;
  • Registered user:
    • Author;
    • Respondent.

An anonymous user is not registered on the web site and cannot take a survey. A registred user authenticates against the local user database (in this case against MySQL) and can be of two types. First, an author is the one which creates surveys and writes the questions. He can save his data, edit or delete it. Second, a respondent is a registered user which can answer the questions or complete some surveys.

Requirements

Functional requirements

We are going to define the requirements as a list of actions which should be implemented and accessible for a end user.

  • User authentication:
    • User logs in;
    • User logs out;
  • Writing surveys:
    • Create a survey;
    • List all created surveys;
    • Select a survey with identifier;
    • Edit selected survey;
    • Delete a survey;
    • Display results of a survey;
    • Writing questions:
      • Create a question - specify the type, may be possible answers;
      • Modify a question;
      • Delete a question;
      • Assign a question to a survey / group of questions;
      • Remove a question from a survey / group of questions;
      • Display results of a question;
  • Filling a survey:
    • Fill a question;
    • Fill a group of question (optional);
    • Display filled values of a respondent;

Non-functional requirements

Non-functional requirements represent the technologies and frameworks behind our Survey service. These are summarized below:

  • PHP programming language;
  • MySQL database for storing data;
  • Web-based user interface:
    • HTML;
    • CSS;
  • Application deployment on the server webdev.fit.cvut.cz.
